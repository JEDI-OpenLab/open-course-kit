Préparer une ressource ouverte

Avant de publier : nettoyer, documenter, clarifier

Une ressource pédagogique ouverte doit pouvoir être comprise, reprise et adaptée par d’autres. Avant de la publier, il faut donc vérifier ce qu’elle contient, ce qui peut être partagé, ce qui doit être cité, ce qui doit être retiré et ce qui mérite d’être mieux documenté.

Cette page propose une méthode simple pour transformer un dossier de travail, un support de cours ou un ensemble de fichiers en ressource pédagogique prête à être publiée.

Point de vigilance

Préparer une ressource ouverte ne consiste pas à tout mettre en ligne. Il faut distinguer les contenus originaux, les contenus tiers, les données personnelles, les documents confidentiels et les éléments qui restent soumis à des droits ou à des règles institutionnelles.

Une méthode en six étapes

1

Inventorier

Lister les fichiers, supports, images, données, exemples, consignes, modèles, scripts, documents sources et éléments annexes qui composent la ressource.

2

Trier

Séparer ce qui est original, ce qui vient de tiers, ce qui peut être partagé, ce qui doit être cité, ce qui doit être remplacé et ce qui ne doit pas être publié.

3

Nettoyer

Retirer les données sensibles, anonymiser les exemples, supprimer les documents internes, vérifier les images et remplacer les contenus dont les droits ne sont pas clairs.

4

Choisir les formats

Prévoir une version facile à consulter et, lorsque c’est utile, une version source modifiable : Markdown, HTML, ODT, PPTX, CSV, SVG ou tout autre format réutilisable.

5

Documenter

Ajouter un titre clair, une description, un public cible, des objectifs pédagogiques, un mode d’emploi, une licence, des crédits et des conseils de réutilisation.

6

Relire

Vérifier que la ressource est compréhensible hors contexte, que les droits sont explicités et qu’une autre personne peut raisonnablement la reprendre sans vous demander d’explications supplémentaires.

1. Faire l’inventaire des fichiers

La première étape consiste à regarder le dossier tel qu’il existe réellement. Une ressource pédagogique contient souvent plus d’éléments qu’on ne l’imagine : supports de cours, consignes, images, exports, brouillons, documents sources, fichiers de travail, scripts, bibliographies, captures d’écran ou exemples étudiants.

Exemple de grille d’inventaire
Élément Type Origine Publier ? Action à prévoir
Support de cours Diaporama Création originale Oui Ajouter licence et version PDF
Illustrations Images Sources diverses À vérifier Contrôler les droits ou remplacer
Travaux étudiants Exemples Production étudiante Non par défaut Anonymiser, demander autorisation ou retirer
Bibliographie Références Sources citées Oui Normaliser les références

2. Identifier les contenus originaux

Les contenus originaux sont les éléments que vous avez produits vous-même ou que votre équipe a produits dans un cadre permettant leur publication : textes, consignes, gabarits, exemples, schémas, fichiers web, jeux de données pédagogiques, activités ou scénarios.

À faire

Pour chaque contenu original, indiquez clairement la licence choisie. Open Course Kit est distribué sous licence MIT. D’autres ressources peuvent retenir une licence différente selon leur objectif de réutilisation.

3. Repérer les contenus tiers

Un contenu tiers est un élément que vous n’avez pas créé vous-même ou dont les droits ne vous appartiennent pas entièrement. Il peut être utilisable, mais il ne peut pas être automatiquement placé sous la même licence que vos propres contenus.

  • Images trouvées en ligne, photographies, pictogrammes, captures d’écran
  • Extraits de livres, articles, rapports, pages web ou documents institutionnels
  • Logos, marques, identités visuelles, noms de produits ou services
  • Polices, bibliothèques, scripts, plugins, composants logiciels
  • Documents sources importés dans un corpus, une base de connaissances ou un outil RAG
  • Productions étudiantes, témoignages, verbatims, données d’enquête

Bonne pratique

Lorsque les droits d’un contenu tiers ne sont pas clairs, il vaut mieux le remplacer, le retirer, le citer sans le republier, ou renvoyer vers la source originale plutôt que de l’intégrer directement dans une ressource ouverte.

4. Vérifier les données personnelles et sensibles

Les ressources pédagogiques contiennent parfois des éléments qui n’ont pas vocation à être diffusés : noms d’étudiants, adresses e-mail, copies, notes, commentaires, photos de groupe, verbatims, résultats d’enquête, échanges internes ou documents institutionnels non publics.

À retirer ou anonymiser

Noms, coordonnées, numéros étudiants, photos identifiables, notes, commentaires individualisés, copies, productions étudiantes non autorisées ou informations permettant d’identifier une personne.

À vérifier avant publication

Données d’enquête, verbatims, exemples issus de situations réelles, documents internes, captures d’écran d’outils, échanges de mails, supports comportant des informations institutionnelles sensibles.

5. Choisir des formats réutilisables

Une ressource vraiment réutilisable ne doit pas seulement être consultable. Elle doit aussi pouvoir être adaptée. Lorsque c’est possible, proposez à la fois une version de consultation et une version source modifiable.

Formats utiles pour publier une ressource pédagogique ouverte
Besoin Format de consultation Format source conseillé
Guide ou documentation HTML, PDF Markdown, HTML, ODT
Support de présentation PDF PPTX, ODP, Markdown slides
Tableau ou jeu de données HTML, PDF CSV, XLSX, JSON
Schéma ou illustration PNG, JPG SVG, fichier source éditable
Webapp ou démonstrateur Site publié HTML, CSS, JavaScript, README

6. Documenter la ressource

Une ressource ouverte doit pouvoir être comprise par quelqu’un qui ne connaît pas le contexte initial. La documentation n’est donc pas un supplément : c’est ce qui rend la réutilisation possible.

Contexte

Indiquer le public visé, le niveau, la durée, les prérequis, le matériel nécessaire et le contexte d’usage initial.

Usage

Expliquer comment utiliser la ressource : déroulé, consignes, étapes, modalités d’animation, variantes possibles.

Réutilisation

Préciser la licence, les crédits, les sources, les limites, les contenus tiers et les conditions de citation.

7. Proposer une arborescence simple

Une structure claire facilite la reprise. Même pour une petite ressource, quelques dossiers bien nommés permettent de comprendre rapidement ce qui relève de la documentation, des sources, des images ou des fichiers diffusables.

Exemple d’arborescence

ma-ressource-ouverte/
├── README.md
├── LICENSE
├── CITATION.cff
├── docs/
│   ├── index.md
│   └── guide-animation.md
├── assets/
│   ├── images/
│   └── fichiers/
├── sources/
│   └── fichiers-modifiables/
└── credits/
    └── sources-et-droits.md

Checklist avant publication

Avant de publier, cette checklist permet de vérifier les points essentiels. Elle peut être copiée dans un README, une issue GitHub ou un document de suivi.

Modèle copiable

## Checklist avant publication

- [ ] Les auteurs et contributeurs sont identifiés.
- [ ] Les contenus originaux sont distingués des contenus tiers.
- [ ] La licence des contenus originaux est indiquée.
- [ ] Les images, icônes, captures d’écran et médias ont été vérifiés.
- [ ] Les extraits textuels et citations sont correctement attribués.
- [ ] Les données personnelles ont été retirées ou anonymisées.
- [ ] Les documents internes ou confidentiels ont été retirés.
- [ ] Les marques, logos et services tiers sont signalés comme tels.
- [ ] Une section “Crédits et sources” est présente.
- [ ] Un fichier LICENSE est présent.
- [ ] Un README.md décrit le projet et ses conditions de réutilisation.
- [ ] Les fichiers sont organisés dans une arborescence compréhensible.
- [ ] Les formats sources modifiables sont fournis quand c’est pertinent.
- [ ] Les consignes sont compréhensibles hors contexte.
- [ ] Les liens principaux ont été testés.

Erreurs fréquentes à éviter

Mettre une licence sur tout sans vérifier

Une licence ouverte ne couvre pas automatiquement les contenus tiers intégrés à la ressource. Il faut signaler les exceptions.

Publier uniquement un PDF fermé

Un PDF peut être utile pour consulter, mais il est souvent insuffisant pour permettre une adaptation facile de la ressource.

Oublier le contexte d’usage

Sans public cible, durée, objectifs ou consignes, une ressource devient difficile à reprendre par quelqu’un d’autre.

Laisser des données personnelles

Les noms, notes, copies, photos et productions étudiantes doivent faire l’objet d’une attention particulière avant publication.

À retenir

Préparer une ressource ouverte, c’est rendre explicite ce qui est souvent implicite : qui a produit quoi, sous quelle licence, avec quelles sources, dans quel contexte et pour quels usages possibles. Plus la ressource est claire, plus elle devient réellement réutilisable.
